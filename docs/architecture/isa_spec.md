# Sirius GPGPU ISA 规范

## 1. 概述

本文档定义 Sirius GPGPU 的完整指令集架构（ISA）。

### 1.1 设计原则

- **全量IR指令适配**：完整覆盖LLVM IR对应的6大类核心操作
- **指令类型分层设计**：核心基线指令（必选）+ 扩展特性指令（可选）
- **寄存器设计兼容并行模型**：通用寄存器支持标量/向量统一访问
- **完全贴合OpenCL标准**：原生支持标量/向量运算，完整覆盖内存模型
- **解码逻辑极简兼容扩展**：固定指令格式，仅通过opcode区分指令类型

### 1.2 指令格式

统一32位固定长度指令：

```
[31:24] opcode    8位操作码
[23:0]  operands  24位操作数
```

操作数格式根据指令类型分为三种：
1. **三寄存器格式**：8位Rd + 8位Rs1 + 8位Rs2
2. **寄存器+立即数/地址格式**：8位Rd + 16位立即数/地址
3. **多字段功能位格式**：8位Rd + 8位Rs1 + 4位功能位 + 4位保留

## 2. 指令集详细定义

### 2.1 算术运算指令

| opcode | 指令名称 | 指令功能 | 操作数格式 | 对应OpenCL操作 | 类型 |
|--------|----------|----------|------------|----------------|------|
| 0x01 | ADD.F32 | 单精度浮点数加法：Rd = Rs1 + Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | float a + float b | 必选 |
| 0x02 | SUB.F32 | 单精度浮点数减法：Rd = Rs1 - Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | float a - float b | 必选 |
| 0x03 | MUL.F32 | 单精度浮点数乘法：Rd = Rs1 × Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | float a × float b | 必选 |
| 0x04 | DIV.F32 | 单精度浮点数除法：Rd = Rs1 ÷ Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | float a ÷ float b | 必选 |
| 0x05 | ADD.I32 | 32位有符号整数加法：Rd = Rs1 + Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | int a + int b | 必选 |
| 0x06 | SUB.I32 | 32位有符号整数减法：Rd = Rs1 - Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | int a - int b | 必选 |
| 0x07 | MUL.I32 | 32位有符号整数乘法：Rd = Rs1 × Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | int a × int b | 必选 |
| 0x08 | NEG.F32 | 单精度浮点数取反：Rd = -Rs1 | 8位Rd + 8位Rs1 + 8位保留（0x00） | -float a | 可选 |
| 0x09 | NEG.I32 | 32位有符号整数取反：Rd = -Rs1 | 8位Rd + 8位Rs1 + 8位保留（0x00） | -int a | 可选 |
| 0x0A | REM.I32 | 32位有符号整数模运算：Rd = Rs1 % Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | int a % int b | 可选 |
| 0x0B | DIV.U32 | 32位无符号整数除法：Rd = Rs1 ÷ Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | uint a ÷ uint b | 可选 |

### 2.2 逻辑运算指令

| opcode | 指令名称 | 指令功能 | 操作数格式 | 对应OpenCL操作 | 类型 |
|--------|----------|----------|------------|----------------|------|
| 0x10 | AND | 32位按位逻辑与：Rd = Rs1 & Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | int a & int b | 必选 |
| 0x11 | OR | 32位按位逻辑或：Rd = Rs1 \| Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | int a \| int b | 必选 |
| 0x12 | XOR | 32位按位逻辑异或：Rd = Rs1 ^ Rs2 | 8位Rd + 8位Rs1 + 8位Rs2 | int a ^ int b | 必选 |
| 0x13 | NOT | 32位按位逻辑非：Rd = ~Rs1 | 8位Rd + 8位Rs1 + 8位保留（0x00） | ~int a | 必选 |
| 0x14 | ICMP | 32位有符号整数比较：Rd=1（真）/0（假） | 8位Rd + 8位Rs1 + 8位（4位比较类型+4位保留） | a>b/a<b/a==b等 | 必选 |
| 0x15 | FCMP | 单精度浮点数比较：Rd=1（真）/0（假） | 8位Rd + 8位Rs1 + 8位（4位比较类型+4位保留） | a>b/a<b/a==b等 | 必选 |
| 0x16 | ICMP.U | 32位无符号整数比较：Rd=1（真）/0（假） | 8位Rd + 8位Rs1 + 8位（4位比较类型+4位保留） | uint a>uint b等 | 可选 |
| 0x17 | FCMP.NAN | 单精度浮点数非数判断：Rd=1（是NAN）/0（非NAN） | 8位Rd + 8位Rs1 + 8位保留（0x00） | isnan(float a) | 可选 |

**比较类型功能位定义**：
- 0001 = 大于（>）
- 0010 = 小于（<）
- 0011 = 等于（==）
- 0100 = 大于等于（≥）
- 0101 = 小于等于（≤）
- 0110 = 不等于（≠）
- 其余 = 保留为0

### 2.3 内存访问指令

| opcode | 指令名称 | 指令功能 | 操作数格式 | 对应OpenCL操作 | 类型 |
|--------|----------|----------|------------|----------------|------|
| 0x20 | LD.GLOBAL | 从全局内存加载数据：Rd = [Rs1 + 偏移量] | 8位Rd + 8位Rs1（基地址） + 8位偏移量 | __global *a[i] | 必选 |
| 0x21 | LD.LOCAL | 从局部内存加载数据：Rd = [Rs1 + 偏移量] | 8位Rd + 8位Rs1（基地址） + 8位偏移量 | __local *a[i] | 必选 |
| 0x22 | LD.PRIVATE | 从私有内存加载数据：Rd = [Rs1 + 偏移量] | 8位Rd + 8位Rs1（基地址） + 8位偏移量 | 局部变量存取 | 必选 |
| 0x23 | LD.CONST | 从常量内存加载数据：Rd = [Rs1 + 偏移量] | 8位Rd + 8位Rs1（基地址） + 8位偏移量 | __constant *a[i] | 必选 |
| 0x24 | ST.GLOBAL | 向全局内存存储数据：[Rs1 + 偏移量] = Rs2 | 8位Rs1（基地址） + 8位Rs2（数据） + 8位偏移量 | __global *a[i] = val | 必选 |
| 0x25 | ST.LOCAL | 向局部内存存储数据：[Rs1 + 偏移量] = Rs2 | 8位Rs1（基地址） + 8位Rs2（数据） + 8位偏移量 | __local *a[i] = val | 必选 |
| 0x26 | ST.PRIVATE | 向私有内存存储数据：[Rs1 + 偏移量] = Rs2 | 8位Rs1（基地址） + 8位Rs2（数据） + 8位偏移量 | 局部变量赋值 | 必选 |
| 0x27 | GEP | 地址计算：Rd = Rs1 + Rs2 × 数据位宽 | 8位Rd + 8位Rs1（基地址） + 8位Rs2（索引） | 数组索引地址计算 | 必选 |
| 0x28 | LD.GLOBAL.STRIDE | 全局内存跨步加载：Rd = [Rs1 + Rs2 × 步长] | 8位Rd + 8位Rs1（基地址） + 8位Rs2（步长） | 向量数据批量加载 | 可选 |

### 2.4 控制流指令

| opcode | 指令名称 | 指令功能 | 操作数格式 | 对应OpenCL操作 | 类型 |
|--------|----------|----------|------------|----------------|------|
| 0x30 | BR.COND | 条件分支：Rs1为1则跳转到地址Rs2，否则顺序执行 | 8位Rs1（条件） + 16位Rs2（目标地址） | if-else判断 | 必选 |
| 0x31 | BR.UNCOND | 无条件分支：跳转到地址Rs1 | 8位保留（0x00） + 16位Rs1（目标地址） | 循环跳转 | 必选 |
| 0x32 | RET | 函数返回：结束当前核函数执行 | 24位保留（0x000000） | 核函数结束 | 必选 |
| 0x33 | BR.LOOP | 循环分支优化：自动递减计数器Rs1，不为0则跳转到Rs2 | 8位Rs1（计数器） + 16位Rs2（目标地址） | for循环优化 | 可选 |

**phi节点适配说明**：无需设计专用opcode，通过硬件在分支合并处维护寄存器上下文实现——当分支跳转至合并点时，硬件自动根据跳转来源选择对应的寄存器值写入目标寄存器，等同于phi节点的功能。

### 2.5 并行相关指令

| opcode | 指令名称 | 指令功能 | 操作数格式 | 对应OpenCL操作 | 类型 |
|--------|----------|----------|------------|----------------|------|
| 0x40 | GET_GLOBAL_ID | 获取全局线程ID：Rd = get_global_id(Rs1) | 8位Rd + 8位Rs1（维度：0/1/2） + 8位保留 | get_global_id(0) | 必选 |
| 0x41 | GET_LOCAL_ID | 获取局部线程ID：Rd = get_local_id(Rs1) | 8位Rd + 8位Rs1（维度：0/1/2） + 8位保留 | get_local_id(0) | 必选 |
| 0x42 | GET_GLOBAL_SIZE | 获取全局网格尺寸：Rd = get_global_size(Rs1) | 8位Rd + 8位Rs1（维度：0/1/2） + 8位保留 | get_global_size(0) | 必选 |
| 0x43 | GET_LOCAL_SIZE | 获取局部工作组尺寸：Rd = get_local_size(Rs1) | 8位Rd + 8位Rs1（维度：0/1/2） + 8位保留 | get_local_size(0) | 必选 |
| 0x44 | GET_GROUP_ID | 获取工作组ID：Rd = get_group_id(Rs1) | 8位Rd + 8位Rs1（维度：0/1/2） + 8位保留 | get_group_id(0) | 必选 |
| 0x45 | GET_NUM_GROUPS | 获取工作组数量：Rd = get_num_groups(Rs1) | 8位Rd + 8位Rs1（维度：0/1/2） + 8位保留 | get_num_groups(0) | 必选 |
| 0x46 | BARRIER_LOCAL | 局部内存屏障：同步工作组内所有线程 | 24位保留（0x000000） | barrier(CLK_LOCAL_MEM_FENCE) | 必选 |
| 0x47 | BARRIER_GLOBAL | 全局内存屏障：同步工作组内所有线程 | 24位保留（0x000000） | barrier(CLK_GLOBAL_MEM_FENCE) | 可选 |

## 3. 寄存器设计

### 3.1 通用寄存器（GPR）

- 数量：32个通用寄存器（R0-R31）
- 位宽：32位
- 用途：支持标量/向量统一访问
- R0：硬连线为0
- R1-R30：通用
- R31：链接寄存器（可选，用于函数调用）

### 3.2 专用ID寄存器

- `GLOBAL_ID_0`、`GLOBAL_ID_1`、`GLOBAL_ID_2`：全局线程ID（各维度）
- `LOCAL_ID_0`、`LOCAL_ID_1`、`LOCAL_ID_2`：局部线程ID（各维度）
- `GLOBAL_SIZE_0`、`GLOBAL_SIZE_1`、`GLOBAL_SIZE_2`：全局网格尺寸（各维度）
- `LOCAL_SIZE_0`、`LOCAL_SIZE_1`、`LOCAL_SIZE_2`：局部工作组尺寸（各维度）
- `GROUP_ID_0`、`GROUP_ID_1`、`GROUP_ID_2`：工作组ID（各维度）
- `NUM_GROUPS_0`、`NUM_GROUPS_1`、`NUM_GROUPS_2`：工作组数量（各维度）

## 5. 实现优先级

### Phase 1：最小可行集（必选指令）
- 算术运算：0x01-0x07（7条）
- 逻辑运算：0x10-0x15（6条）
- 内存访问：0x20-0x27（8条）
- 控制流：0x30-0x32（3条）
- 并行相关：0x40-0x46（7条）
- **总计**：31条必选指令

### Phase 2：可选指令扩展
- 算术运算：0x08-0x0B（4条）
- 逻辑运算：0x16-0x17（2条）
- 内存访问：0x28（1条）
- 控制流：0x33（1条）
- 并行相关：0x47（1条）
- **总计**：9条可选指令
