# Sirius GPGPU 模块接口定义

## 1. 取指模块（Fetch Unit）

### 1.1 输入信号

| 信号名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| clk | 1 | 输入 | 时钟信号 |
| rst_n | 1 | 输入 | 异步复位（低有效） |
| branch_target | 32 | 输入 | 分支目标地址 |
| branch_taken | 1 | 输入 | 分支是否跳转 |
| icache_rdata | 128 | 输入 | I-Cache读数据（4指令） |
| icache_ready | 1 | 输入 | I-Cache就绪 |

### 1.2 输出信号

| 信号名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| pc | 32 | 输出 | 当前PC值 |
| icache_addr | 32 | 输出 | I-Cache地址 |
| icache_en | 1 | 输出 | I-Cache使能 |
| inst0 | 32 | 输出 | 指令0 |
| inst1 | 32 | 输出 | 指令1 |
| inst2 | 32 | 输出 | 指令2 |
| inst3 | 32 | 输出 | 指令3 |
| pc0 | 32 | 输出 | 指令0的PC |
| pc1 | 32 | 输出 | 指令1的PC |
| pc2 | 32 | 输出 | 指令2的PC |
| pc3 | 32 | 输出 | 指令3的PC |

## 2. 译码模块（Decode Unit）

### 2.1 输入信号

| 信号名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| clk | 1 | 输入 | 时钟信号 |
| rst_n | 1 | 输入 | 异步复位（低有效） |
| inst | 32 | 输入 | 32位指令 |
| pc | 32 | 输入 | 指令PC值 |
| rs1_rdata | 32 | 输入 | 源寄存器1读数据 |
| rs2_rdata | 32 | 输入 | 源寄存器2读数据 |
| rs1_ready | 1 | 输入 | 源寄存器1就绪 |
| rs2_ready | 1 | 输入 | 源寄存器2就绪 |

### 2.2 输出信号

| 信号名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| rs1_addr | 5 | 输出 | 源寄存器1地址 |
| rs2_addr | 5 | 输出 | 源寄存器2地址 |
| rd_addr | 5 | 输出 | 目标寄存器地址 |
| rs1_data | 32 | 输出 | 源寄存器1数据 |
| rs2_data | 32 | 输出 | 源寄存器2数据 |
| imm | 32 | 输出 | 立即数 |
| uop_opcode | 6 | 输出 | 微指令操作码 |
| uop_type | 3 | 输出 | 微指令类型（ALU/Load/Store/Branch） |
| uop_pc | 32 | 输出 | 微指令PC |
| uop_valid | 1 | 输出 | 微指令有效 |
