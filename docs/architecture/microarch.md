# Sirius GPGPU 微架构设计

## 1. 概述

本文档定义 Sirius GPGPU 的微架构设计。

### 1.1 设计目标

- **简化超标量**：2发射超标量，降低复杂度
- **简化乱序执行**：重排序缓冲区（ROB）+ 保留站（RS）
- **缓存分级**：L1指令缓存 + L1数据缓存 + L2共享缓存
- **简化互连**：片上网络（NoC）简化版本

### 1.2 流水线阶段

```
IF  -- 取指
ID  -- 译码
IS  -- 发射
EX  -- 执行
MEM -- 访存
WB  -- 写回
```

共6级流水线。

---

## 2. 核心模块拆分

### 2.1 取指模块（Fetch Unit）

**职责**：
- 从指令缓存取指
- PC生成与维护
- 分支预测

**输入**：
- 分支目标地址（来自执行模块）
- 分支预测结果

**输出**：
- 32位指令（到译码模块）
- PC值（到译码模块）

---

### 2.2 译码模块（Decode Unit）

**职责**：
- 指令译码
- 寄存器重命名
- 操作数读取

**输入**：
- 32位指令
- PC值

**输出**：
- 译码后的微指令（uOP）
- 源寄存器操作数
- 目标寄存器

---

### 2.3 发射模块（Issue Unit）

**职责**：
- 保留站管理
- 指令发射（2发射超标量）
- 依赖检查

**输入**：
- 译码后的微指令
- 源寄存器操作数

**输出**：
- 发射到执行单元的指令
- 操作数

---

### 2.4 执行模块（Execute Unit）

**职责**：
- 算术逻辑单元（ALU）
- 浮点运算单元（FPU）
- 分支执行
- 条件码生成

**子模块**：
- **整数ALU**：整数加减乘除、逻辑运算
- **浮点FPU**：浮点加减乘除
- **分支单元**：分支跳转、条件检查
- **比较单元**：整数/浮点数比较

**输入**：
- 发射的指令
- 操作数

**输出**：
- 执行结果
- 分支目标地址
- 条件码

---

### 2.5 访存模块（Memory Unit）

**职责**：
- 加载/存储队列管理
- 缓存访问
- 内存序维护

**子模块**：
- **加载队列（LQ）**：管理加载操作
- **存储队列（SQ）**：管理存储操作
- **D-Cache**：L1数据缓存

**输入**：
- 访存指令
- 地址
- 数据（存储操作）

**输出**：
- 数据（加载操作）
- 缓存状态

---

### 2.6 写回模块（Writeback Unit）

**职责**：
- 结果写回寄存器堆
- ROB提交
- 异常处理

**输入**：
- 执行结果
- 访存结果

**输出**：
- 写回寄存器的数据
- ROB提交信号

---

## 3. 关键数据结构

### 3.1 重排序缓冲区（ROB）

- **大小**：32条目
- **职责**：
  - 维护指令顺序
  - 异常处理
  - 提交指令

### 3.2 保留站（RS）

- **大小**：16条目
- **职责**：
  - 等待操作数就绪
  - 指令发射

### 3.3 加载队列（LQ）

- **大小**：16条目
- **职责**：
  - 管理加载操作
  - 地址冲突检测

### 3.4 存储队列（SQ）

- **大小**：16条目
- **职责**：
  - 管理存储操作
  - 延迟提交到缓存

---

## 4. 缓存架构

### 4.1 L1指令缓存（I-Cache）

- **大小**：16KB
- **行大小**：64字节
- **关联度**：4路组相联
- **命中时间**：1周期

### 4.2 L1数据缓存（D-Cache）

- **大小**：16KB
- **行大小**：64字节
- **关联度**：4路组相联
- **命中时间**：2周期
- **写策略**：写回+写分配

### 4.3 L2共享缓存

- **大小**：128KB
- **行大小**：64字节
- **关联度**：8路组相联
- **命中时间**：10周期
- **包含**：包含L1缓存内容

---

## 5. 多计算单元（CU）设计

### 5.1 CU数量

- **基础配置**：4个计算单元
- **可扩展**：最多16个计算单元

### 5.2 SIMT多线程

- **每个CU线程数**：64个工作项（Work-Item）
- **Warp大小**：16个工作项
- **Barrier同步**：硬件支持
