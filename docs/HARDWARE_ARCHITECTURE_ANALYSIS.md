## 二、SIMT 架构设计

### 2.1 SIMT 架构原理

**SIMT (Single-Instruction Multiple-Thread)** - 单指令多线程架构

Sirius 采用类似 NVIDIA CUDA 的 SIMT 架构：
- 多个线程执行同一条指令
- 每个线程有独立的寄存器上下文
- 线程以 Warp（线程束）为单位调度
- 支持分支 divergence（分支发散）

### 2.2 Sirius SIMT 实现方案

#### 2.2.1 线程管理
```
线程层次结构：
├── Grid（网格）
│   ├── Work-Group（工作组）
│   │   ├── Warp（线程束，32个线程）
│   │   │   ├── Thread 0
│   │   │   ├── Thread 1
│   │   │   ├── ...
│   │   │   └── Thread 31
│   │   └── ...
│   └── ...
└── ...
```

#### 2.2.2 专用 ID 寄存器
Sirius ISA 提供专用指令获取线程信息：

| 指令 | 功能 | 对应 OpenCL 内置函数 |
|------|------|---------------------|
| `GET_GLOBAL_ID` | 获取全局线程 ID | `get_global_id(dim)` |
| `GET_LOCAL_ID` | 获取局部线程 ID | `get_local_id(dim)` |
| `GET_GLOBAL_SIZE` | 获取全局网格尺寸 | `get_global_size(dim)` |
| `GET_LOCAL_SIZE` | 获取局部工作组尺寸 | `get_local_size(dim)` |
| `GET_GROUP_ID` | 获取工作组 ID | `get_group_id(dim)` |
| `GET_NUM_GROUPS` | 获取工作组数量 | `get_num_groups(dim)` |
| `BARRIER_LOCAL` | 局部内存屏障 | `barrier(CLK_LOCAL_MEM_FENCE)` |

#### 2.2.3 Warp 调度机制
- **Warp 大小**：32 个线程
- **调度粒度**：按 Warp 调度
- **分支处理**：使用分支掩码（branch mask）处理 divergence
- **同步机制**：`BARRIER_LOCAL` 实现工作组内同步

### 2.3 OpenCL 内存模型支持

| 内存类型 | Sirius 支持 | 指令 |
|---------|------------|------|
| 全局内存（__global） | ✅ | `LD.GLOBAL`, `ST.GLOBAL` |
| 局部内存（__local） | ✅ | `LD.LOCAL`, `ST.LOCAL` |
| 私有内存（__private） | ✅ | `LD.PRIVATE`, `ST.PRIVATE` |
| 常量内存（__constant） | ✅ | `LD.CONST` |

## 三、SIMD 并行计算

### 3.1 SIMD 原理

**SIMD (Single-Instruction Multiple-Data)** - 单指令多数据

Sirius 内部使用 SIMD 并行计算：
- 一条指令同时处理多个数据元素
- 支持向量运算（float32/int32）
- 标量/向量统一访问的寄存器设计

### 3.2 Sirius SIMD 实现

#### 3.2.1 寄存器设计
- **通用寄存器（GPR）**：支持标量/向量统一访问
- **向量宽度**：设计预留 128 位（4×float32 或 4×int32）
- **标量 fallback**：向量指令可降级为标量执行

#### 3.2.2 SIMD 指令支持
虽然当前 ISA 主要定义标量指令，但架构预留了 SIMD 扩展：

| 扩展方向 | 说明 |
|---------|------|
| 向量算术 | `VADD.F32`, `VSUB.F32`, `VMUL.F32`, `VDIV.F32` |
| 向量逻辑 | `VAND`, `VOR`, `VXOR` |
| 向量比较 | `VICMP`, `VFCMP` |
| 向量内存 | `VLD.GLOBAL`, `VST.GLOBAL`（跨步访问） |
| 数据置换 | `VSHUFFLE`, `VPERMUTE` |

#### 3.2.3 跨步加载优化
- `LD.GLOBAL.STRIDE` - 全局内存跨步加载
- 支持向量数据批量加载
- 提升内存带宽利用率

## 四、硬件架构梳理

### 4.1 顶层架构（sirius_top）

```
sirius_top
├── 控制接口
│   ├── clk, rst_n
│   ├── start_addr, start, done
│   └── 状态机（IDLE → RUN → DONE）
├── 取指模块（fetch）
│   ├── PC 生成
│   ├── 指令获取
│   └── 内存接口仲裁
├── 译码模块（decode）
│   ├── 指令译码
│   ├── 寄存器读
│   └── 操作数准备
├── 执行模块（execute）
│   ├── ALU（算术逻辑单元）
│   └── 结果写回
├── 访存模块（memory）
│   ├── 加载/存储
│   └── 地址计算
├── 寄存器堆（regfile）
│   ├── 32×32 位通用寄存器
│   └── 双读口单写口
└── 内存接口
    ├── mem_addr, mem_wdata, mem_wstrb
    ├── mem_we, mem_rdata
    └── mem_req, mem_ack
```

### 4.2 模块详解

#### 4.2.1 取指模块（fetch.sv）
**功能**：
- 生成 PC（程序计数器）
- 从内存获取指令
- 处理分支跳转

**关键特性**：
- 支持 `start_addr` 配置起始地址
- 指令流控（valid/ready 握手）
- 简化的分支预测（静态）

#### 4.2.2 译码模块（decode.sv）
**功能**：
- 解析 32 位固定格式指令
- 生成控制信号
- 读取寄存器操作数

**指令格式**：
```
[8位 opcode][24位操作数]
├── 寄存器型：[8位 Rd][8位 Rs1][8位 Rs2]
├── 立即数型：[8位 Rd][16位 立即数]
└── 分支型：[8位 条件][16位 目标地址]
```

**Opcode 映射**：
| 范围 | 指令类型 |
|------|---------|
| 0x00-0x0F | 算术运算 |
| 0x10-0x1F | 逻辑运算 |
| 0x20-0x2F | 内存访问 |
| 0x30-0x3F | 控制流 |
| 0x40-0x4F | 并行相关 |

#### 4.2.3 执行模块（execute.sv）
**功能**：
- 执行算术和逻辑运算
- 处理条件分支
- 生成结果

**ALU 操作**：
- 整数运算：ADD, SUB, MUL, DIV, REM, NEG
- 浮点运算：ADD.F32, SUB.F32, MUL.F32, DIV.F32, NEG.F32
- 逻辑运算：AND, OR, XOR, NOT
- 比较运算：ICMP, FCMP

#### 4.2.4 访存模块（memory.sv）
**功能**：
- 执行加载/存储操作
- 地址计算（GEP）
- 处理四种内存空间

**内存访问模式**：
- LD.GLOBAL/ST.GLOBAL - 全局内存
- LD.LOCAL/ST.LOCAL - 局部内存
- LD.PRIVATE/ST.PRIVATE - 私有内存
- LD.CONST - 常量内存

#### 4.2.5 寄存器堆（regfile.sv）
**规格**：
- 32 个 32 位通用寄存器
- 双读口，单写口
- 寄存器 0 硬连线为 0

**寄存器约定**：
| 寄存器 | 用途 |
|--------|------|
| R0 | 零寄存器（只读为 0） |
| R1-R29 | 通用寄存器 |
| R30 | 线程 ID 基址 |
| R31 | 特殊用途 |

#### 4.2.6 集成模块（integration.sv）
**功能**：
- 模块间连接
- 内存仲裁
- 全局控制

**内存仲裁策略**：
- 简化：优先取指
- 可扩展：多端口内存或交叉开关

## 五、进阶特性实现

### 5.1 简化版超标量
- **设计**：预留双发射接口
- **实现**：当前单发射，可扩展
- **限制**：无结构性冒险检查

### 5.2 简化版乱序执行
- **设计**：预留重排序缓冲区（ROB）接口
- **实现**：当前顺序执行
- **限制**：无 Tomasulo 算法

### 5.3 简化版缓存分级
- **设计**：预留 L1/L2 缓存接口
- **实现**：当前无缓存，直接访存
- **扩展**：cache.sv 框架已预留

### 5.4 简化版复杂互连
- **设计**：预留网络-on-chip (NoC) 接口
- **实现**：当前简单总线
- **扩展**：多 CU 互连

**验证属性**：

**安全性属性**：
- 无越界内存访问
- 无未定义指令执行
- 无死锁（无循环等待）

**活性属性**：
- 程序最终终止（RET 可达）
- 屏障最终释放
- 线程最终同步

**不变量**：
- R0 始终为 0
- PC 对齐（4 字节）
- 寄存器文件状态一致性
